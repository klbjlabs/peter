git 使用技巧：小步快跑

Peter 研究过的 git 技巧挺多，但是真正每天用到的技巧其实少的可怜。今天要分享的就是 Peter 每天调式 html 的时候每天都会敲很多遍的技巧。所有机器服务一个思想，总结起来八个字：小步快跑，方便回滚。具体涉及的技巧有设置别名加快操作，git diff/status 查看修改，回滚历史命令，用 rebase 的方式合并多个小版本成为一个大版本。

别名设置

Peter 的 .gitconfig 文件内容如下

[user]
  name = Peter Wang
  email = happypeter1983@gmail.com
[core]
  editor = vim
[alias]
  ci = commit -a -v
  st = status -s
  br = branch
  throw = reset --hard HEAD
  throwh = reset --hard HEAD^
[color]
  ui = true
[push]
  default = current

项目回滚

采用 git throwh/throw 来扔掉不想要的修改

git diff/status 随时查看都改动了什么内容。
